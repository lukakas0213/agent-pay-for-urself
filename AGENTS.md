# Agent Instructions

## Project

This repository is for `agent-pay-for-urself`.

## Working Guidelines

- Keep changes small and focused on the requested task.
- Read the existing code and configuration before editing.
- Prefer project-local patterns over introducing new frameworks or abstractions.
- Do not overwrite user changes or revert unrelated work.
- Use clear names and simple control flow.

## Validation

- Run the most relevant checks available for the files you change.
- If no automated checks exist yet, verify changes with a focused manual inspection.
- Report any checks that could not be run.

## Git

- Do not create commits unless explicitly asked.
- Do not run destructive git commands such as `git reset --hard` or `git checkout --` unless explicitly asked.
