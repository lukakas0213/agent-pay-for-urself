# System Goal

## 현재 목표

프론트와 백엔드 구조를 아래 5개 화면 중심 제품 구조로 재정렬한다.

* `대시보드`: 메인 에이전트와 ChatGPT처럼 대화하는 채팅 허브
* `에이전트`: 메인 에이전트와 각 서브 에이전트의 프롬프트, 특화 설정 관리
* `히스토리 타임라인`: 메인 에이전트와 서브 에이전트의 실행 순서, 상태, 재실행 흐름 표시
* `보고서`: 날짜순 보고서 목록과 상세 보기
* `계좌`: Toss API, 한국투자 API 기반 계좌 연결과 계좌 상태 확인

## 현재 구현과 목표 차이

### 1. 대시보드

목표:
메인 에이전트와 대화하는 채팅 중심 화면만 둔다.

현재:
실행 폼, 최근 런 카드, 에이전트 요약 카드, 보고서 저장 패널이 함께 있다.

상태:
`부분 구현`

### 2. 에이전트

목표:
메인 에이전트와 각 서브 에이전트의 프롬프트와 에이전트별 특화 설정을 수정한다.

현재:
서브 에이전트 프롬프트 수정은 가능하지만, 메인 에이전트 전용 진입점과 에이전트별 설정 저장 계층은 없다.

상태:
`부분 구현`

### 3. 히스토리 타임라인

목표:
메인 에이전트와 서브 에이전트가 어떻게 일했는지 시간축으로 보여준다.

현재:
전용 화면과 전용 backend contract가 없다.

상태:
`미구현`

### 4. 보고서

목표:
날짜순으로 보고서를 리스트업하고 클릭하면 상세를 본다.

현재:
저장된 보고서 목록과 상세 조회는 가능하다.

상태:
`대체로 구현됨`

### 5. 계좌

목표:
Toss API, 한국투자 API 계좌 연결 입력과 현재 연결된 계좌 상태를 제공한다.

현재:
`GET /account` 조회는 있으나, 연결 정보는 주로 브라우저 로컬 상태에 머물고 Toss 연결 계약은 없다.

상태:
`부분 구현`

| Toss 계좌 연결 | 필요 | 미완료 | adapter와 API 계약 필요 |

## 백엔드 기준 구현 여부

### 이미 있는 것

* `POST /decisions`
* `POST /console/interactions`
* `GET /experiments`
* `GET /experiments/{experiment_id}`
* `POST /experiments/from-run`
* `GET /agent-prompts`
* `PUT /agent-prompts/{agent_key}`
* `GET /account`

### 추가로 필요한 것

* 메인 에이전트를 포함한 프롬프트 목록 표준화
* 에이전트별 특화 설정 저장/조회 API
* run 목록 API
* run 상세 타임라인 API
* 재실행 분기 이력 기록 구조
* Toss 계좌 연결 계약
* 계좌 연결 정보 저장/조회 API
* `GET /account`의 connection, credential status 정합성 보강

## 구현 순서

1. 대시보드를 메인 에이전트 채팅 허브로 단순화한다.
2. 에이전트 화면에 메인 에이전트와 에이전트별 설정 계층을 추가한다.
3. 히스토리 타임라인용 run list/detail backend contract를 추가한다.
4. 보고서 화면을 날짜순 리스트와 상세 보기 중심으로 고정한다.
5. 계좌 연결을 한국투자와 Toss 기준으로 재설계한다.
6. 테스트, API 계약 문서, 프론트 사용 문서를 동기화한다.
