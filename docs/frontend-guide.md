# 프론트 사용 안내

이 문서는 이 저장소의 프론트 화면을 처음 보는 사람도 바로 따라할 수 있게 만든 사용 안내서다.

## 한눈에 보기

프론트는 크게 5가지 화면으로 구성된다.

* `메인화면`: 메인 에이전트 채팅, 자동매매 승인 스위치, 실행 타임라인, 주문 전 보고서를 함께 보는 허브
* `계좌 상태`: 연결할 계좌 정보를 적고, 현재 보유 종목과 수익률을 카드/표 형태로 보는 곳
* `보고서`: 저장한 실행 결과를 다시 열어 주문 전 보고서와 최종 판단 흐름을 복기하는 곳
* `에이전트`: 단계별 출력과 개별 프롬프트를 확인하는 상세 화면
* `프롬프트 설정`: 에이전트 프롬프트와 메인화면 기본값을 조정하는 곳

## 시작 전 준비

프론트를 쓰려면 보통 아래 2가지만 준비하면 된다.

* 브라우저
* 백엔드 서버가 실행 중인 상태

백엔드가 꺼져 있으면 화면은 열려도 실행, 저장, 계좌 조회가 되지 않을 수 있다.

## 1. 메인화면에서 실행하고 대화하기

메인화면은 이제 단순 입력 폼이 아니라 메인 에이전트 허브다. 한 화면에서 아래 흐름을 따라간다.

* 채팅으로 메인 에이전트에 자연어 지시를 보낸다.
* 자동매매 승인 스위치를 확인한다.
* 심볼, 최대 비중, 목표를 정하고 실행한다.
* 오른쪽 상태 카드와 하단 타임라인에서 에이전트 상태를 본다.
* 주문 전 보고서를 검토하고 필요하면 저장한다.

### 메인 에이전트 채팅

채팅창에는 아래 같은 내용을 바로 보낼 수 있다.

* 관찰 종목을 추가해 달라는 요청
* 특정 종목 비중을 더 보수적으로 보라는 요청
* 어떤 리스크를 우선 설명해 달라는 요청

실행 결과가 이미 있으면 채팅은 그 결과를 설명하거나, 설정에 따라 후속 지시로 반영해 워크플로우를 다시 돌릴 수 있다.

### 자동매매 승인 스위치

상단 상태 카드의 `자동매매 승인` 스위치는 현재 브라우저 기본값으로 저장된다.

* 켜짐: 라이브 주문 승인 필요 플래그를 해제하는 방향으로 메인 화면 기본값을 잡는다.
* 꺼짐: 수동 승인 필요 상태로 유지한다.

현재 API는 주문 계획만 반환하며, 이 화면이 실제 주문을 전송하지는 않는다.

### 실행할 때 입력하는 내용

* `심볼`: 확인할 종목 코드
* `최대 비중`: 한 종목에 허용할 최대 비중
* `메인 목표`: 메인 에이전트가 이번 실행에서 우선 해석할 자연어 목표
* `허용 심볼`: 허용하고 싶은 종목만 적는다.
* `제외 심볼`: 빼고 싶은 종목만 적는다.
* `리스크 성향`: 낮음, 보통, 높음 중 하나를 고른다.
* `추가 메모`: 기타 제약을 남긴다.

### 타임라인과 상태 보기

메인화면 중간의 타임라인에는 아래 이벤트가 순서대로 쌓인다.

* 워크플로우 실행 시작/완료/실패
* 메인 에이전트 채팅 답변
* 후속 지시 반영 재실행
* 보고서 저장 성공/실패

오른쪽 상태 패널에는 각 에이전트가 `러닝중`, `연결됨`, `연결안됨` 중 어떤 상태인지 표시된다.

### 주문 전 보고서 보기

하단 보고서 영역에서는 먼저 아래 내용을 확인한다.

* 데이터 수집 개수
* 데이터 분석 개수
* 보고서 작성 결과
* 주문 계획 개수

종목별 카드에는 다음이 담긴다.

* 보고서 요약
* 상승 포인트
* 주의 포인트
* 리스크 플래그
* 추천 액션 편향

## 2. 계좌 상태에서 보유 종목 보기

`계좌 상태` 화면은 계좌 연결 입력과 보유 종목 현황을 함께 보여준다.

### 입력 가능한 내용

* 계좌 별칭
* 브로커 이름
* 계좌 번호
* 상품 코드

현재 저장소 계약에는 계좌 연결 정보를 저장하는 서버 API가 없으므로, 이 값은 브라우저 로컬 설정으로만 저장된다.

### 볼 수 있는 내용

* 예수금
* 총매입금액
* 총평가금액
* 총수익률
* 종목별 카드형 요약
* 종목별 수량, 평균단가, 현재가, 평가금액, 손익, 수익률 표

## 3. 보고서에서 저장된 실행 복기하기

`보고서` 화면은 저장한 실행 기록을 모아 다시 보는 곳이다.

### 볼 수 있는 내용

* 저장한 보고서 목록
* 실행 시각과 런타임
* 메인 에이전트 요약
* 주문 전 보고서 승인 개수
* 입력 목표와 채팅 지시
* 종목별 보고서, 최종 판단, 주문 계획

### 사용하는 방법

1. 왼쪽 목록에서 보고 싶은 보고서를 고른다.
2. 오른쪽에서 실행 단계와 주문 전 보고서를 먼저 확인한다.
3. 필요하면 수집 데이터와 최종 판단까지 내려가며 비교한다.

## 4. 에이전트 메뉴에서 단계별 점검하기

상단 메뉴의 `에이전트`를 누르면 단계별 상세 화면으로 이동할 수 있다.

현재 단계는 다음과 같다.

* `데이터 수집`
* `데이터 분석`
* `보고서 작성`
* `매수/매도 판단`
* `주문 실행`
* `로그/평가`

각 화면에서는 저장된 프롬프트와 최근 실행 결과를 같이 볼 수 있다.

## 5. 프롬프트 설정에서 기본값 조정하기

`프롬프트 설정` 화면에서는 아래 두 가지를 관리한다.

* 에이전트별 프롬프트 본문 저장
* 메인화면 기본 심볼, 리스크 성향, 타임라인 보관 수, 자동매매 승인 기본값 저장

이 기본값은 현재 브라우저 로컬 스토리지에 저장되며, 메인화면과 계좌 상태 화면이 함께 사용한다.

## 문제가 있을 때

* 화면이 열리지 않으면 백엔드가 먼저 켜져 있는지 확인한다.
* 이전 버전 브라우저 저장 결과나 일부 내부 필드명(`trade_decisions`, `order_plans`)이 남아 있어도 메인화면, 보고서, 에이전트 화면은 현재 계약(`decisions`, `orders`)에 맞게 자동 보정해 연다.
* 실행 버튼이 안 눌리면 심볼과 최대 비중 형식을 확인한다.
* 채팅 후 재실행이 안 되면 먼저 한 번 실행해 `run_id`를 만든다.
* 계좌가 비어 있으면 브로커 환경 변수와 계좌 연결 상태를 확인한다.
* 저장이 안 되면 보고서 이름이 비어 있지 않은지 확인한다.
