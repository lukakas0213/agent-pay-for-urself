# 프론트 사용 안내

이 문서는 현재 프론트 화면 구조와 실제 사용 흐름을 빠르게 따라갈 수 있게 정리한 운영 안내서다.

## 한눈에 보기

프론트는 고정 사이드바, 상단 헤더, 카드형 대시보드 구조를 기준으로 아래 5개 화면을 제공한다.

* `메인화면`: 워크플로우 실행, 후속 대화, 최근 런 복기, 보고서 저장을 한 화면에서 처리하는 메인 콘솔
* `계좌 상태`: `GET /account` 응답을 예수금, 총평가금액, 보유 종목 카드와 표로 보는 화면
* `보고서`: 저장된 실행 결과를 목록과 상세 패널로 나눠 복기하는 화면
* `에이전트`: 단계별 프롬프트와 최근 실행 결과를 함께 점검하는 화면
* `프롬프트 설정`: 에이전트 프롬프트와 브라우저 로컬 워크스페이스 기본값을 관리하는 화면

## 시작 전 준비

프론트를 사용하려면 아래 두 가지가 필요하다.

* 브라우저
* 실행 중인 백엔드 서버

백엔드가 꺼져 있으면 사이드바의 상태 패널이 연결 오류를 표시하고, 실행/저장/계좌 조회가 실패할 수 있다.

## 1. 공통 레이아웃 이해하기

모든 화면은 같은 셸을 사용한다.

* 왼쪽 사이드바: 메인 메뉴, 워크플로우 단계, 시스템 링크, 백엔드 헬스 배지
* 상단 헤더: 검색 입력, 워크스페이스 선택, `새 실행` 버튼
* 메인 콘텐츠: 화면별 카드, 표, 입력 폼

사이드바 하단 시스템 패널은 `GET /health` 기준의 단순 연결 상태를 보여준다.

## 2. 메인화면에서 워크플로우 실행하기

메인화면은 `POST /decisions`, `POST /console/interactions`, `POST /experiments/from-run`을 직접 사용하는 운영 대시보드다.

### 실행 입력

실행 패널에서 아래 값을 입력한다.

* `심볼`
* `최대 비중`
* `리스크 성향`
* `메인 목표`
* `허용 심볼`
* `제외 심볼`
* `추가 메모`
* `자동매매 승인 기본값`
* `후속 대화를 워크플로우에 반영`

`실행` 버튼을 누르면 현재 입력으로 메인 워크플로우를 돌리고, 응답 결과를 현재 런과 최근 런 카드에 저장한다.

### 실행 후 확인하는 영역

실행이 끝나면 같은 화면에서 아래를 바로 본다.

* `현재 선택된 실행`: run id, 런타임, 결정 수, 주문 계획 수, 제약 위반 수
* `최근에 본 워크플로우 런`: 브라우저에 저장된 최근 런 카드 목록
* `자주 보는 에이전트 뷰`: 데이터 수집, 분석, 보고서, 판단, 주문, 평가 요약 카드
* `워크플로우 그룹 프리뷰`: 단계 연결 미리보기 카드

최근 런 카드를 누르면 그 실행이 현재 선택 상태로 올라온다.

### 후속 대화

후속 대화 패널은 현재 선택된 `run_id`를 기준으로 `POST /console/interactions`를 호출한다.

* 질문만 할 때: 실행 결과 설명 응답만 받는다.
* `후속 대화를 워크플로우에 반영`이 켜져 있을 때: 같은 메시지를 후속 지시로 붙여 재실행하고 결과를 갱신한다.

최근 6개의 대화는 카드 형태로 유지된다.

### 보고서 저장

저장 패널은 현재 선택된 실행을 `POST /experiments/from-run`으로 저장한다.

입력값은 아래 두 가지다.

* `보고서 이름`
* `설명`

저장이 성공하면 보고서 화면에서 같은 실행을 다시 열 수 있다.

## 3. 계좌 상태 화면 보기

`계좌 상태` 화면은 계좌 입력 폼과 실계좌 스냅샷을 함께 보여준다.

### 로컬로 저장하는 값

아래 입력값은 현재 브라우저 로컬 스토리지에만 저장한다.

* `계좌 별칭`
* `브로커`
* `계좌 번호`
* `상품 코드`

현재 저장소 계약에는 이 값을 저장하는 별도 서버 API가 없다.

### API에서 받는 값

`GET /account` 응답은 아래 영역에 바인딩된다.

* 요약 카드: 예수금, 총매입금액, 총평가금액, 총수익률
* 보유 종목 카드: 평가금액, 비중, 손익, 수익률
* 상세 표: 수량, 평균단가, 현재가, 평가금액, 손익, 수익률

## 4. 보고서 화면에서 저장 실행 복기하기

`보고서` 화면은 좌측 목록과 우측 상세 패널로 나뉜다.

* 좌측 목록: `GET /experiments`
* 우측 상세: `GET /experiments/{experiment_id}`

상세 패널에서는 아래 순서로 복기한다.

1. 메인 에이전트 요약과 런타임 확인
2. 입력과 제약 확인
3. 주문 전 보고서 확인
4. 최종 판단과 주문 계획 확인
5. 수집 데이터 확인

## 5. 에이전트 화면에서 단계별 점검하기

에이전트 허브와 상세 화면은 아래 단계를 기준으로 이동한다.

* `데이터 수집`
* `데이터 분석`
* `보고서 작성`
* `매수/매도 판단`
* `주문 실행`
* `로그/평가`

각 상세 화면은 두 가지를 같이 보여준다.

* 현재 저장된 프롬프트
* 최근 메인화면 실행 결과에서 해당 단계가 만든 출력 카드

## 6. 프롬프트 설정 화면 사용하기

프롬프트 설정 화면은 프롬프트 목록과 편집 패널, 로컬 기본값 패널로 나뉜다.

### 서버에 저장하는 값

아래는 프롬프트 API를 사용한다.

* 목록 조회: `GET /agent-prompts`
* 개별 저장: `PUT /agent-prompts/{agent_key}`

### 브라우저 로컬에 저장하는 값

아래 기본값은 현재 브라우저 워크스페이스 상태로만 저장한다.

* 기본 심볼
* 기본 최대 비중
* 기본 리스크 성향
* 타임라인 보관 수
* 후속 채팅 자동 반영 여부
* 자동매매 승인 기본값
* 계좌 별칭
* 브로커 이름

## 문제가 있을 때

* 사이드바 상태 패널이 오류면 먼저 백엔드가 켜져 있는지 확인한다.
* 메인화면 실행이 안 되면 심볼과 최대 비중 형식을 확인한다.
* 후속 대화가 재실행을 못 하면 먼저 한 번 실행해 `run_id`를 만든다.
* 보고서 저장이 안 되면 이름이 비어 있지 않은지 확인한다.
* 계좌가 비어 있으면 브로커 환경 변수와 계좌 연결 상태를 확인한다.
* 이전 버전 브라우저 저장 결과에 `trade_decisions`, `order_plans`가 남아 있어도 프론트는 현재 계약인 `decisions`, `orders` 기준으로 자동 보정해 읽는다.
