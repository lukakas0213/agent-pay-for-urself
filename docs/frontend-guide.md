# 프론트 사용 안내

이 문서는 이 저장소의 프론트 화면을 처음 보는 사람도 바로 따라할 수 있게 만든 사용 안내서다.

## 한눈에 보기

프론트는 크게 3가지 화면으로 구성된다.

* `메인화면`: 종목을 넣고 워크플로우를 실행하는 곳
* `계좌`: 모의투자 계좌의 보유 종목과 손익을 보는 곳
* `보고서`: 저장한 실행 결과를 다시 보는 곳
* `에이전트`: 허브에서 세부 목차로 나누어 각 단계의 결과를 자세히 보는 곳

## 시작 전 준비

프론트를 쓰려면 보통 아래 2가지만 준비하면 된다.

* 브라우저
* 백엔드 서버가 실행 중인 상태

백엔드가 꺼져 있으면 화면은 열려도 실행이나 저장이 되지 않을 수 있다.

## 1. 메인화면에서 실행하기

메인화면은 워크플로우를 시작하는 출발점이다.

### 입력하는 내용

* `심볼`: 확인하고 싶은 종목 코드
  * 예: `AAPL, MSFT`
  * 쉼표로 여러 종목을 넣을 수 있다.
* `최대 비중`: 한 종목에 얼마나 크게 투자할지 정하는 값
  * 예: `0.2`은 20%를 뜻한다.
* `목표`: 이번 실행의 목적
  * 예: "보수적으로 미국 주식 후보를 검토한다."
* `허용 심볼`: 허용하고 싶은 종목만 적는다.
* `제외 심볼`: 빼고 싶은 종목만 적는다.
* `리스크 성향`: 낮음, 보통, 높음 중 하나를 고른다.
* `라이브 주문 승인 필요`: 실제 주문으로 이어질 때 추가 확인이 필요한지 정한다.
* `추가 메모`: 참고할 내용이 있으면 적는다.

### 실행 방법

1. 종목과 조건을 입력한다.
2. `실행` 버튼을 누른다.
3. 잠시 기다리면 결과가 아래에 표시된다.

### 실행 후 보는 내용

실행이 끝나면 아래 순서로 결과를 확인할 수 있다.

* 어떤 종목을 불러왔는지
* 데이터가 어떻게 해석됐는지
* 위험 기준에 걸렸는지
* 최종적으로 매수, 매도, 보류 중 무엇으로 판단했는지
* 실제 제출 가능한 주문인지
* 전체 실행 요약이 어떤지

### 결과 저장

실행 결과를 나중에 다시 보고 싶으면 저장할 수 있다.

1. `보고서 이름`을 적는다.
2. 필요하면 `설명`도 적는다.
3. `저장` 버튼을 누른다.
4. 저장이 끝나면 `보고서` 메뉴에서 확인할 수 있다.

저장하지 않아도 방금 실행한 결과는 같은 브라우저 안에서 잠시 남아 있을 수 있다.

## 2. 계좌에서 보유 종목 보기

`계좌` 화면은 한국투자 모의투자 계좌를 읽어서 현재 보유 종목과 손익을 확인하는 곳이다.

### 볼 수 있는 내용

* 예수금
* 총평가금액
* 총손익
* 총수익률
* 종목별 보유 수량, 평균단가, 현재가, 평가금액, 손익, 수익률

### 사용하는 방법

1. 상단 메뉴에서 `계좌`를 누른다.
2. 잠시 기다리면 계좌 요약과 보유 종목 표가 표시된다.
3. `조회 불가`가 보이면 모의투자 계좌 설정이나 브로커 환경 변수를 확인한다.

## 3. 보고서에서 다시 보기

`보고서` 화면은 저장한 실행 기록을 모아 보는 곳이다.

### 볼 수 있는 내용

* 저장한 보고서 목록
* 실행 시각
* 입력한 종목
* 실행 당시의 판단 결과
* 각 에이전트가 만든 요약 내용

### 사용하는 방법

1. 왼쪽 목록에서 보고 싶은 보고서를 고른다.
2. 오른쪽에서 상세 내용을 확인한다.
3. 입력값, 과정, 결과를 한 번에 비교할 수 있다.

## 4. 에이전트 메뉴에서 세부 화면으로 이동하기

상단 메뉴의 `에이전트`를 먼저 선택하면 에이전트 허브가 열린다. 여기서 서브 목차를 골라 단계별 상세 화면으로 이동할 수 있다.

각 화면이 뜻하는 바는 다음과 같다.

* `데이터 수집`: 종목의 시세, 뉴스, 재무 지표를 모은다.
* `데이터 분석`: 모은 정보를 점수와 근거로 정리한다.
* `리스크 관리`: 주문이 허용되는지 확인한다.
* `매수/매도 판단`: 최종 행동을 결정한다.
* `주문 실행`: 실제 주문 전에 제출 가능 여부와 수량을 정리한다.
* `로그/평가`: 전체 실행을 요약한다.

에이전트 허브와 상세 화면에서는 최근 실행 결과를 바탕으로 각 단계의 출력값을 자세히 볼 수 있다.

## 자주 쓰는 예시

### 예시 1: 두 종목만 간단히 확인하기

* 심볼: `AAPL, MSFT`
* 최대 비중: `0.2`
* 목표: `보수적으로 검토`

이렇게 넣고 실행하면 두 종목 기준으로 전체 흐름이 돌아간다.

### 예시 2: 특정 종목은 빼고 보기

* 심볼: `AAPL, MSFT, NVDA`
* 제외 심볼: `NVDA`

이렇게 하면 제외한 종목은 판단 대상에서 빠진다.

### 예시 3: 더 조심스럽게 보기

* 리스크 성향: `낮음`
* 라이브 주문 승인 필요: 켬

이렇게 하면 결과를 더 보수적으로 확인하는 데 도움이 된다.

## 결과를 볼 때 기억할 점

* 숫자는 참고용 해석값이며, 자동으로 투자 성과를 보장하지 않는다.
* `실행 완료`와 `실제 주문`은 같은 뜻이 아니다.
* 보고서에 저장된 내용은 실행 당시의 기록이다.
* 브라우저를 닫기 전 저장하지 않으면 일부 내용은 다시 찾기 어려울 수 있다.

## 문제가 있을 때

* 화면이 열리지 않으면 백엔드가 먼저 켜져 있는지 확인한다.
* 실행 버튼이 안 눌리면 종목이 비어 있지 않은지 확인한다.
* 저장이 안 되면 보고서 이름이 비어 있지 않은지 확인한다.
* 결과가 비어 있으면 한 번 실행한 뒤 다시 확인한다.
