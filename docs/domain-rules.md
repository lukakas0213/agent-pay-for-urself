# 도메인 규칙

## 투자 시스템 원칙

### 반드시 지켜야 할 사항

* 투자 근거 없는 매수 금지
* 리스크 검증 없는 주문 금지
* 단일 지표 기반 투자 금지
* 백테스트 없는 전략 배포 금지
* 성과 평가 없는 전략 유지 금지

### 투자 판단 흐름

데이터 수집

→ 데이터 분석

→ 리스크 검증

→ 매수/매도 판단

→ 주문 실행

→ 성과 평가

현재 최소 구현 기준으로는 위 순서를 기본 계약으로 본다.
포트폴리오 검토 단계는 향후 별도 에이전트 또는 정책 계층으로 확장할 수 있다.

## 금지 사항

절대 수행하지 말 것

* API Key 하드코딩
* 비밀번호 저장
* 테스트 삭제
* 무단 스키마 변경
* 사용되지 않는 대규모 리팩토링
* 검증되지 않은 투자 전략 배포
* 리스크 검증 우회
* 실거래 환경에서 테스트 코드 실행
* 실험실 prompt override로 출력 스키마 또는 리스크 검증 우회

## 실험실 안전 경계

* 실험실은 전략, 모델, 프롬프트 검사용 UI이며 기본적으로 실제 주문 제출을 허용하지 않는다.
* `EXPERIMENT_LIVE_ORDER_ENABLED=true`가 설정된 경우에만 실험실 결과에서 주문 제출 가능 상태를 허용할 수 있다.
* 환경변수로 제출 가능 상태가 허용되어도 실제 broker submit은 별도 adapter 구현과 승인 경계가 있어야 한다.
* 실험 기록에는 API key, broker secret, 인증 토큰 같은 민감 정보를 저장하지 않는다.

## 프로젝트 도메인 지식

### 투자 시스템

* 미국 주식 중심
* 기본 브로커는 한국투자증권 Open API 사용
* 미국 주식 주문은 한국투자증권 해외주식 주문 API 기준으로 설계
* 데이터 기반 투자
* 설명 가능한 투자 판단
* 리스크 우선 설계

### 멀티 에이전트

* LangGraph 기반
* Agent Orchestrator 중심 구조
* Agent 역할 분리
* Agent 출력 표준화
